# 天气工具循环调用使用说明

## 1. 作业内容

程序文件：`week11_weather_tool_loop.py`

本作业将老师原来的天气查询拆分成两个工具：

- `get_coordinates(city)`：根据城市名称查询经纬度。
- `get_weather(latitude, longitude)`：根据经纬度查询天气。

程序使用循环 Function Call。查询城市天气时，大模型会先调用坐标工具，再使用返回的经纬度调用天气工具，最后生成回答。

## 2. 安装依赖

在终端执行：

```bash
python3 -m pip install openai httpx
```

- `openai`：调用兼容 OpenAI 协议的大模型。
- `httpx`：调用 Open-Meteo 坐标和天气接口。

## 3. 配置大模型

在 macOS 或 Linux 终端执行：

```bash
export OPENAI_API_KEY="你的API Key"
export OPENAI_BASE_URL="https://ai.btcapp.net/v1"
export OPENAI_MODEL="gpt-5.6-sol"
```

API Key 只设置在本机环境变量中，不要写入代码或上传到 Git。

## 5. 查询示例

只查询城市坐标：

```bash
python3 week11_weather_tool_loop.py -q "上海的经纬度是多少？"
```

根据城市查询天气：

```bash
python3 week11_weather_tool_loop.py -q "上海现在天气怎么样？"
```

根据已知坐标直接查询天气：

```bash
python3 week11_weather_tool_loop.py -q "北纬31.22、东经121.46的天气怎么样？"
```

只测试坐标和天气接口，不调用大模型：

```bash
python3 week11_weather_tool_loop.py --self-test
```

## 6. 链式调用过程

```text
用户询问“上海天气”
    ↓
get_coordinates(city="上海")
    ↓
返回 latitude 和 longitude
    ↓
get_weather(latitude, longitude)
    ↓
返回真实天气数据
    ↓
大模型生成最终回答
```

坐标和天气数据来自 Open-Meteo 免费接口，不需要额外的天气 API Key。
